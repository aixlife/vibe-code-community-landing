## 2. 이 프로젝트의 목적

나라장터(국가종합전자조달)를 매일 수동으로 확인하는 반복 업무를 자동화합니다.

**자동화 범위:**
- 신규 입찰공고 수집 및 관심 공고 필터링
- 투찰 이력 구글시트에 자동 기록
- 투찰 결과(낙찰/유찰/개찰대기) 자동 추적
- 공고별 우선순위 분석 및 투찰 추천

## 3. 데이터 소스

- **공공데이터포털**: https://www.data.go.kr (API 키 발급 후 사용, 무료)
- **이력 관리 구글시트**: {시트URL}

> ⚠️ **업무구분 주의**: 각 API의 오퍼레이션이 업무 유형별(물품/용역/공사/외자)로 분리되어 있습니다.
> 대표님 업종에 맞는 오퍼레이션만 호출해야 정상 응답이 옵니다.
> 예) IT인프라 → "용역" 오퍼레이션 / 건설·전기공사 → "공사" 오퍼레이션

### API 키 발급 방법 (처음 시작하는 경우)

1. https://www.data.go.kr 접속 → 회원가입 (무료)
2. 검색창에 "나라장터 입찰공고" 입력
3. 원하는 API 서비스 클릭 → "활용신청" 버튼
4. 신청 후 1~2일 내 발급 (자동 승인인 경우 즉시 발급)
5. 마이페이지 → 개발계정 → 인증키 확인

### 🔴 나라장터 직접 크롤링 금지

나라장터 직접 크롤링은 이용약관 위반 시 민법 제750조 불법행위 책임 소지가 있습니다.
**반드시 공공데이터포털 API만 사용하세요.** API로 동일한 데이터를 법적 리스크 없이 가져올 수 있습니다.

## 4. AI 응답 원칙 (반드시 지킬 것)

1. **코딩 설명 최소화**: 대표님은 개발자가 아닙니다. 코드 블록은 복붙용으로만 제공하고, 설명은 **무엇을 하는지 / 어떤 결과가 나오는지**에 집중하세요.
2. **단계별 확인**: 자동화 실행 전, 항상 "이렇게 진행할게요. 진행할까요?" 형식으로 확인을 받으세요.
3. **결과는 표로 시각화**: 구글시트, 마크다운 표, 또는 CSV로 항상 눈에 보이게 정리.
4. **에러는 해결책부터**: 에러가 나면 원인 분석보다 "이렇게 해결하면 됩니다" 먼저 제시.
5. **법적 이슈 사전 확인**: 나라장터 크롤링 전 `robots.txt`, 이용약관 반드시 확인. 불가능하면 공공데이터포털 API 사용 유도.
6. **한국어로 응답**: 모든 답변은 한국어로.

## 5. 사용할 도구

- **Antigravity** (또는 Claude Code) — 이 프로젝트의 AI 에이전트
- **구글 스프레드시트** — 이력 및 분석 저장소
- **Google Apps Script** (선택) — 시트 자동 업데이트
- **공공데이터포털 API** — 안정적인 데이터 소스 (크롤링 대신 권장)

## 6. 프롬프트 사용 순서

1. `prompts/01-collect-and-filter.md` — 공고 수집 & 관심 공고 필터링
2. `prompts/02-bid-history.md` — 투찰 이력 구글시트 기록
3. `prompts/03-result-tracking.md` — 낙찰/유찰 결과 추적
4. `prompts/04-summary-analysis.md` — 공고 요약 & 우선순위 분석

각 프롬프트는 독립 실행 가능하나, **01 → 02 → 03 → 04 순서로 한 번씩 돌리면 전체 파이프라인이 완성**됩니다.
