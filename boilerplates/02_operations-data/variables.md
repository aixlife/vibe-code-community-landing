## 분석 축 선택 가이드

신청서의 "어떤 분석이 가장 필요하신가요?" 답변에 따라 프롬프트 우선순위를 조정하세요.

| 니즈 | 추천 프롬프트 순서 |
|------|-----------------|
| 매출 현황 파악이 우선 | 01 → 02 → 04 |
| 특정 날 매출이 이상했다 | 01 → 03 |
| 메뉴 정리/가격 조정 검토 | 01 → 02 (메뉴별 분석 요청) |
| 월말 결산 보고서 필요 | 04 → 02 |
| 인건비 최적화 | 01 (시간대 분석) → 02 |

## 주요 분석 축 (POS 데이터에서 뽑을 수 있는 것)

| 분석 축 | 알 수 있는 것 | 활용 |
|--------|------------|------|
| 시간대별 매출 | 피크타임 / 비수기 시간 | 직원 배치 최적화 |
| 요일별 매출 | 주중/주말 패턴 | 발주량·재고 계획 |
| 메뉴별 매출 | 베스트셀러 / 사라져야 할 메뉴 | 메뉴 구성 조정 |
| 객단가 | 한 테이블 평균 소비 | 세트 메뉴·업셀링 전략 |
| 결제 수단 | 현금/카드/배달 비율 | 수수료 관리 |
| 이상치 | 평소와 다른 날 | 원인 분석·재발 방지 |

## 다중 매장 설정

매장이 2개 이상이면 CLAUDE.md에 매장별 정보를 추가하세요:

```
- 매장 1: {매장1명} / POS: {매장1POS} / 시트탭: {매장1탭}
- 매장 2: {매장2명} / POS: {매장2POS} / 시트탭: {매장2탭}
```

프롬프트 실행 시 "어느 매장 데이터인지" 상단에 명시하면 분석이 분리됩니다.

## CSV 컬럼 자동 매핑

POS마다 컬럼명이 다를 수 있습니다. 아래 동의어를 참고해 AI가 자동으로 인식합니다.

| 의미 | 인식하는 컬럼명 예시 |
|------|------------------|
| 날짜 | 날짜, 일자, date, 거래일 |
| 시간 | 시간, 거래시간, time |
| 메뉴명 | 메뉴, 상품명, 품목, item |
| 수량 | 수량, qty, 개수 |
| 단가 | 단가, 가격, price |
| 합계 | 합계, 금액, 결제금액, amount, total |
| 결제수단 | 결제, 결제방법, payment |
