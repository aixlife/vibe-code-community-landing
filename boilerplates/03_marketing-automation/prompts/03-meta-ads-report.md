# 프롬프트 03: 메타 광고 성과 자동 리포트 (축 C)

> 대행사 대표님의 핵심 프롬프트입니다. 여러 클라이언트의 광고 성과를 한 번에 리포트화합니다.
> 자사 브랜드 운영자도 동일하게 사용 가능.

---

## 사전 확인 (실행 전 체크)

- [ ] Meta for Developers에서 액세스 토큰을 발급받았다
- [ ] 광고 계정 ID(act_XXXXXXXXXX)를 알고 있다
- [ ] 리포트 기간을 정했다 (주간 / 월간)

---

## 프롬프트 본문

```
당신은 {회사명}의 광고 성과 분석을 담당하는 AI 에이전트입니다.

[기본 정보]
- 운영 형태: {운영형태}  <!-- 자사 브랜드 / 마케팅 대행사 -->
- 리포트 기간: {리포트기간}  <!-- 예: 2026-04-14 ~ 2026-04-20 (지난주) -->
- 리포트 수신자: {수신자}  <!-- 예: 내부용, 클라이언트 이메일 발송 -->

[광고 계정 정보]
- 광고 계정 ID: (환경변수 META_AD_ACCOUNT_ID 에서 읽어옴)
- 액세스 토큰: (환경변수 META_ACCESS_TOKEN 에서 읽어옴)

[대행사 클라이언트 목록 — 대행사이면 채워주세요]
- 클라이언트 1: {클라1명} / 계정 ID: {클라1계정ID}
- 클라이언트 2: {클라2명} / 계정 ID: {클라2계정ID}
<!-- 클라이언트 수만큼 추가 -->

[수집할 지표]
- 필수: 노출수, 클릭수, CTR, CPC, 전환수, ROAS, 광고비 소진액
- 선택: CPM, 빈도, 도달수, 영상 조회수(해당 시)

[작업 순서]
1. Meta Ads API를 통해 위 기간의 광고 성과 데이터를 수집합니다.
2. 아래 형식으로 리포트를 자동 생성합니다:
   - 전체 요약 (한 눈에 보기)
   - 캠페인별 성과 비교표
   - 전주 대비 변화 (▲▼ 표시)
   - AI 인사이트 3가지 (잘된 것 / 개선 필요한 것 / 다음 주 추천 액션)
3. 리포트를 구글시트({시트URL})에 자동 기록합니다.
4. 대행사이면 클라이언트별 분리 리포트를 각각 생성합니다.

[리포트 스타일]
- 수신자가 "클라이언트"이면: 전문 용어 최소화, 핵심 수치만
- 수신자가 "내부용"이면: 상세 지표 전체 포함
- 모든 리포트는 한국어로 작성

시작 전에 "이렇게 진행할게요"로 계획을 먼저 말해주세요.
```

## 출력 형식 예시

### 주간 광고 성과 리포트 (2026.04.14 ~ 04.20)

**전체 요약**
| 지표 | 이번 주 | 전주 | 변화 |
|------|--------|------|------|
| 광고비 소진 | 1,250,000원 | 1,100,000원 | ▲ 13.6% |
| 노출수 | 485,000 | 420,000 | ▲ 15.5% |
| 클릭수 | 3,200 | 2,800 | ▲ 14.3% |
| CTR | 0.66% | 0.67% | ▼ 0.01%p |
| ROAS | 4.2 | 3.8 | ▲ 10.5% |

**AI 인사이트**
1. ✅ 잘된 것: "봄 신제품" 캠페인 ROAS 6.1로 가장 높음
2. ⚠️ 개선 필요: 리타겟팅 광고 CTR이 0.3%로 평균 절반 수준
3. 📌 다음 주 추천: 리타겟팅 소재 교체 + 신제품 예산 20% 증액

---

## 활용 팁

- **월간 리포트**: 기간을 "지난달 1일 ~ 말일"로 변경하면 동일하게 동작합니다.
- **클라이언트 자동 발송**: 리포트 완성 후 "이메일로 발송해줘"를 추가하면 Gmail API 연동이 가능합니다.
- **인사이트 심화**: 리포트 완성 후 "04 GA4 인사이트 프롬프트"를 이어서 실행하면 웹 전환 데이터와 교차 분석됩니다.
