## 신청서 질문 → 변수 매핑

| 신청서 질문 | CLAUDE.md 변수 | 예시 값 |
|-----------|--------------|--------|
| 회사명 / 업종 | `{회사명}`, `{업종}` | 하우스오브비 / 화장품 |
| 자사 브랜드인가, 대행사인가 | `{운영형태}` | 자사 브랜드 / 마케팅 대행사 |
| 현재 운영 중인 마케팅 채널 | `{채널목록}` | 인스타그램, 워드프레스 블로그, 메타 광고 |
| 월 광고 집행 예산 | `{광고예산}` | 월 500만원 |
| 관리하는 클라이언트 수 (대행사) | `{클라이언트수}` | 5개사 / 해당 없음 |
| 인스타그램 비즈니스 계정 여부 | `{비즈니스계정여부}` | 있음 / 없음 |
| 성과 데이터 저장 위치 | `{시트URL}` | https://docs.google.com/... |

---

## 운영 형태별 분기

### 자사 브랜드
- `{클라이언트수}` → "해당 없음"
- 추천 축 우선순위: **B(인스타) → D(소재) → C(광고성과) → A(블로그)**
- 리포트 구조: 단일 브랜드 기준

### 마케팅 대행사
- `{클라이언트수}` → 실제 관리 클라이언트 수 입력
- 추천 축 우선순위: **C(성과리포트) → 통합 캘린더 → B(인스타) → D(소재)**
- 리포트 구조: 클라이언트별 분리 (각 프롬프트 실행 시 클라이언트명 변수 추가)

## 채널별 사전 준비 체크

| 채널 | 필요한 것 | 발급 위치 |
|------|---------|---------|
| Meta 광고 API | 액세스 토큰 | Meta for Developers → My Apps → 토큰 생성 |
| Instagram Graph API | 비즈니스 계정 + 페이지 연결 | Facebook 페이지 → Instagram 계정 연결 |
| WordPress REST API | 앱 비밀번호 | 워드프레스 관리자 → 사용자 → 프로필 → 앱 비밀번호 |
| GA4 | 속성 ID (G-XXXXXXXX) | GA4 관리 → 속성 → 데이터 스트림 |

## 대행사 전용: 클라이언트 변수 추가 방법

프롬프트 실행 시 아래 변수를 추가로 입력하세요:

```
- 클라이언트명: {클라이언트명}
- 클라이언트 업종: {클라업종}
- 클라이언트 광고 계정 ID: {클라계정ID}
- 리포트 수신자: {수신자이메일}
- 리포트 주기: {주기}  <!-- 주간 / 월간 -->
```
